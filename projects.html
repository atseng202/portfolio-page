SIS (Student Information System)

Python
Django
PostgreSQL

SIS is an administrative student information system app built with for staff to manage curriculum at Rithm School and students to obtain important course updates daily. It is built with a Django and PostgreSQL backend.

As an intern, I helped build key administrative features on the backend for staff to use, including administrative actions to download student assessments with proper groupings.

I helped set up a paginated calendar of upcoming lectures, exercises, and events with proper cache invalidation for cohorts. I also integrated Slack notifications with the backend such that when staff uploaded a new curriculum asset, Slack would automatically send notifications about the uploaded material in the associated Slack. Lastly, I helped build a generic editable form page which allowed staff to bulk edit an interdependent list of lectures, exercises, events, and assessments for a cohort. This was done using Django FormSets.

Jobly

JavaScript
React
Express.js
Node.js
PostgreSQL

Jobly is a full stack app that lists employers and matches users to apply for new jobs. It uses Node/Express with PostgreSQL on the backend and React as the frontend.

Users can register / login with their account credentials. Logged in users can browse and filter companies and view jobs to apply for.

I used Test Driven Development (TDD) in the backend, which emphasized a lot of thought into how routes should function and if any middlewares were properly added. I also had to think carefully about how the frontend component hierarchy should be implemented.

Warbler

Python
Flask
Jinja
HTML5
PostgreSQL
SQLAlchemy
jQuery
Axios

Warbler is a server-side Twitter clone app. It uses a Flask backend and renders HTML with Jinja. Users can register / login with their account credentials, edit their profile, write tweets, check out other users' tweets, and like / unlike them. Unit and integration tests were written with the unittest module for models and view functions. Used Bcrypt to store salted password hashes in the database.

I fixed bugs with the logout functionality, user routes, and homepage routes, in addition to adding functionality to edit the user profile and like or unlike tweets with Axios. I also achieved extensive test coverage in our models and views with unit and integration testing.

This project was an opportunity to work with a larger codebase and enforce the habit and importance of writing good tests to maintain a well-functioning codebase that could be expanded upon.

Convocord

JavaScript
React
Redux
Ruby on Rails
Action Cable Web Sockets
PostgreSQL

Convocord is a full stack Discord clone that allows users to join server channels and live chat with friends. It is built with Ruby on Rails and PostgreSQL on the backend, and React/Redux on the frontend. Users can register / login to access protected route features such as adding and joining channels and servers.

This project was a good opportunity to practice designing a backend schema and choosing an appropriate React component hierarchy. It was a fun challenge to figure out how to normalize the data to reduce redundancy with messages attached to server channels and direct message channels. I also had to figure out how to integrate web sockets into the existing code structure without introducing problematic bugs.
